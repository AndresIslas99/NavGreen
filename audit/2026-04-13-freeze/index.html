    Known broken behaviors at freeze time¶

    1. Teleop does not work when a real map is loaded. With has_map=true the safety chain runs collision_monitor → cmd_vel_collision_safe → cmd_vel_gate → cmd_vel_safe → odrive, but cmd_vel_gate has a 0.5s watchdog that forces safety_ok=false before safety_supervisor has published any SafetyStatus. Result: /agv/cmd_vel_safe is published at zero. Dashboard joystick moves have no effect on the ODrive.
    2. Map load does not auto-localize historical maps. Maps saved before 2026-04-13 lack the <name>_cuvslam/, <name>.area, and <name>_meta.json sidecars, so the auto_init_orchestrator cascade falls through Paths A0, A, B, C and reports FAILED. The fix for future maps is already in place (map_manager_node writes all sidecars on Save Map), but retroactively re-saving existing maps is still required.
    3. ros2 daemon cache previously contained ghost sim_* and /teleop nodes from a dev PC at 192.168.55.100 via the l4tbr0 USB bridge peer in cyclonedds.xml. Resolved by setting ROS_DOMAIN_ID=42 in the systemd units — but the <Peer> entry is still hardcoded in the DDS config.

    What is NOT changing during the audit¶

    The audit phases 1-4 are read-only. No code, config, launch file, or systemd unit under src/, /etc/systemd/, or ~/agv_data/ will be modified. The only writes during those phases are: - docs/audit/*.md (new audit reports) - specs/*.yaml (new spec files + updates to existing specs) - tools/verify_specs/* (new verification scripts) - .git/hooks/pre-commit (new git hook) - AGENT_INSTRUCTIONS.md (new root-level doc) - src/*/CLAUDE.md (new / updated per-package docs, no code changes) - src/*/TASK.yaml (new / updated per-package task files)

    Fases 5 and 6 WILL touch src/ code, but only after the baseline from fases 1-4 is reviewed and approved.
